# Godot 原生 MCP 迁移计划 - 执行摘要

## 核心目标
将 Godot-MCP 从 **Node.js/FastMCP 三层架构** 迁移到 **Godot 原生实现（单层架构）**

## 当前架构问题
1. **环境依赖**: 需要 Node.js + npm 安装依赖
2. **通信延迟**: AI → Node.js → Godot 三次序列化
3. **资源开销**: Node.js 进程占用 50-150MB 内存
4. **安全风险**: 命令注入漏洞
5. **代码分散**: TypeScript + GDScript 两套代码

## 目标架构优势
- ✅ **零外部依赖**: 仅需 Godot 引擎
- ✅ **响应延迟降低 30%+**: 消除中间层
- ✅ **内存占用降低 90%+**: 150MB → <5MB
- ✅ **安全性提升**: 直接 API 调用，无命令注入风险
- ✅ **代码统一**: 仅 GDScript

## Claude Desktop 配置变化

### 旧配置 (Node.js)
```json
{
  "mcpServers": {
    "godot-mcp": {
      "command": "node",
      "args": ["PATH/server/dist/index.js"]
    }
  }
}
```

### 新配置 (原生)
```json
{
  "mcpServers": {
    "godot-mcp-native": {
      "command": "path/to/godot.exe",
      "args": ["--path", "PROJECT_PATH", "--headless", "--mcp-server"]
    }
  }
}
```

## 风险评估

| 风险 | 影响 | 概率 | 缓解措施 |
|-----|------|------|---------|
| `OS.read_string_from_stdin()` 阻塞 | 高 | 中 | 使用 Thread 实现非阻塞读取 |
| Godot 头less 模式限制 | 中 | 中 | 提供编辑器模式作为备选 |
| JSON-RPC 实现不完整 | 高 | 低 | 充分测试，参考官方规范 |
